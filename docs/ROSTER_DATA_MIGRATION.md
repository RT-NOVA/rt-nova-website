# Roster Data Migration

Roster pages are now maintained from season-based YAML files, with lightweight markdown route pages generated by script.

## Source of truth

```text
data/seasons/<season-id>/rosters/<team-key>.yaml
```

Examples:

```text
data/seasons/2026/rosters/13u-black.yaml
data/seasons/2026/rosters/13u-orange.yaml
data/seasons/2027/rosters/14u-black.yaml
```

## Generated route files

The route files under `content/rosters/` are generated by:

```bash
python3 scripts/sync-roster-pages.py
```

Examples:

```text
content/rosters/2026-13u-black.md
content/rosters/2026-13u-orange.md
content/rosters/2027-14u-black.md
```

Do not edit those markdown files directly. Edit the YAML data file and rerun the sync script.

## Teams page compatibility

The Teams page configuration is unchanged. Existing links such as `/rosters/2026-13u-orange/` continue to work because the sync script creates the matching route files.

## Add a new roster

Create:

```text
data/seasons/2026/rosters/12u.yaml
```

Then run:

```bash
python3 scripts/sync-roster-pages.py
hugo server -D --disableFastRender
```

## Cleanup after applying

Zip patches cannot delete existing files from your working tree. If needed, run:

```bash
bash scripts/cleanup-roster-transition-files.sh
```

That removes generated Hugo output and obsolete roster-card/photo transition files.


## Term-specific roster pages

Roster data can be split by term when fall and spring players differ. Example:

```text
data/seasons/2026/rosters/11u.yaml
data/seasons/2026/rosters/11u-fall.yaml
```

Use `route_slug` to control the public page URL.


## Fall 2025 imported rosters

The Fall 2025 roster pages are separate from Spring 2026 because player lists can change between terms.

```text
data/seasons/2026/rosters/11u-fall.yaml
data/seasons/2026/rosters/13u-black-fall.yaml
data/seasons/2026/rosters/13u-orange-fall.yaml
```

Team Central Fall 2025 links point to the term-specific roster pages.
